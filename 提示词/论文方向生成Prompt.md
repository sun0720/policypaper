# 论文方向生成 Prompt — 优化版

> 所属模块：`paper-topic-analyzer`
> 原始需求：对每条经济新闻，生成 5 个可落地写作的经济学论文方向
> 优化日期：2026-06-05

---

## 原始提示

> 对每条经济新闻，生成 5 个可落地写作的经济学论文方向

## 识别的问题

### 清晰度
- 「可落地写作」含义模糊 — 什么样的论文方向才算「可落地」？是指数据可获得、方法可行、还是有明确政策含义？
- 「论文方向」未定义粒度 — 是一句话标题、一段摘要、还是包含研究方法和数据来源的完整提案？

### 具体性
- 缺少论文类型的约束 — 实证研究？理论模型？政策评估？文献综述？
- 缺少经济学细分领域的指引 — 宏观、微观、产业经济、国际贸易、公共财政？
- 没有给出期望的输出样例，AI 难以对齐预期

### 完整性
- 未定义成功标准 — 什么样的 5 个方向算「好」？
- 未要求 5 个方向之间的差异性和互补性
- 未说明每条方向的输出应包含哪些要素

### 结构
- 单行提示，缺少结构化分段（背景、任务、约束、输出格式）

### 可操作性
- 有明确动作「生成」，但输出格式未定义，AI 只能猜测

---

## 优化后的 Prompt

```
你是一位经济学研究导师。根据以下经济新闻，为该新闻生成 5 个差异化、可落地的经济学论文方向。

## 输入
一条来自中国政府网（gov.cn）的经济政策/数据新闻。

## 任务要求

### 论文方向定位
- 面向经济学专业研究生或青年学者，论文应具备在 3-6 个月内完成的可操作性
- 以实证研究或政策评估为主（鼓励使用公开数据），也可包含理论建模方向
- 研究问题应从中观层面切入，避免过宽（如「中国经济增长研究」）或过窄

### 5 个方向的差异化要求
每个方向需侧重不同的分析视角，确保多样性：

1. **方向一：因果推断 / 政策效应评估**
   （如 DID、RDD、工具变量、合成控制法）

2. **方向二：机制分析 / 传导路径**
   （如中介效应、行业异质性、空间溢出效应）

3. **方向三：宏观或一般均衡视角**
   （如 DSGE 校准、投入产出分析、CGE 模型）

4. **方向四：微观主体行为**
   （如企业决策、家庭消费、投资者反应、劳动力市场）

5. **方向五：国际比较 / 跨国经验**
   （如与他国类似政策的对比研究、跨国面板数据分析）

### 每条方向的输出必须包含

| 要素 | 说明 |
|------|------|
| 论文标题 | 中英文，学术期刊风格 |
| 研究问题 | 1-2 句话，明确核心因果关系或待验证假设 |
| 研究方法建议 | 具体模型或计量方法，例如 "DID with staggered adoption，使用 A 股上市公司 2018-2024 面板数据" |
| 数据来源建议 | 至少 1 个具体可获取的数据库，如 CSMAR、Wind、CNRDS、国家统计局、CEIC、CGSS、CFPS |
| 政策含义 | 1 句话说明该研究的现实意义 |
| 与新闻的关联 | 1 句话说明该方向如何从新闻出发 |

### 输出格式
对每条新闻，以 Markdown 结构化输出：

## 新闻：[新闻标题]

### 方向一：因果推断 / 政策效应评估
| 要素 | 内容 |
|------|------|
| 论文标题 | ... |
| 研究问题 | ... |
| 研究方法 | ... |
| 数据来源 | ... |
| 政策含义 | ... |
| 与新闻关联 | ... |

（方向二至五同上，依次替换分析视角）

### 质量要求
- 每个方向之间研究问题不重复，互为补充
- 建议的数据来源真实存在且可获取（不要编造数据库）
- 研究方法与问题匹配，避免「用复杂方法回答简单问题」
- 经济学概念使用准确（如正确区分相关与因果）
- 每个方向均有独立的学术贡献点
```

## 在项目中的应用

此 Prompt 用于 `paper-topic-analyzer` 模块，通过 Claude API 调用。

### API 调用建议

```typescript
const systemPrompt = `你是一位经济学研究导师。...（上述完整 prompt）`;

const response = await anthropic.messages.create({
  model: "claude-sonnet-4-6",
  system: systemPrompt,
  messages: [
    {
      role: "user",
      content: `请分析以下经济新闻：\n\n标题：${news.title}\n发布时间：${news.date}\n正文摘要：${news.summary}`,
    },
  ],
  max_tokens: 4096,
});
```

### 与 JSON Schema 输出的关系

本 Prompt 设计为 Markdown 表格输出，适用于直接展示到网站。
如需结构化存储，可追加 JSON Schema 约束：

```json
{
  "news_title": "原文标题",
  "paper_directions": [
    {
      "direction_type": "causal_inference",
      "title_zh": "论文中文标题",
      "title_en": "Paper English Title",
      "research_question": "核心研究问题",
      "methodology": "研究方法建议",
      "data_sources": ["CSMAR", "国家统计局"],
      "policy_implication": "政策含义",
      "news_connection": "与新闻的关联"
    }
  ]
}
```

其中 `direction_type` 枚举值为：
- `causal_inference` — 因果推断/政策效应评估
- `mechanism_analysis` — 机制分析/传导路径
- `macro_ge` — 宏观/一般均衡视角
- `micro_behavior` — 微观主体行为
- `international_comparison` — 国际比较/跨国经验

## 关键改进

| 改进点 | 说明 |
|--------|------|
| ✅ 明确角色定位 | 设定为「经济学研究导师」，锚定输出质量和专业水准 |
| ✅ 5 个方向强制差异化 | 分别指定因果推断、机制分析、宏观视角、微观行为、国际比较，避免方向同质化 |
| ✅ 六要素结构化输出 | 标题 → 研究问题 → 方法 → 数据 → 政策含义 → 关联，每条方向可被直接评估和比较 |
| ✅ 可落地性具象化 | 要求具体数据库名、具体计量方法、3-6 个月时间框架，消除模糊空间 |
| ✅ 表格格式输出 | 便于直接渲染到网站页面 |
| ✅ 内置质量检验 | 方向不重复、数据真实存在、方法-问题匹配，降低 AI 幻觉风险 |
| ✅ API 集成指引 | 提供了 Claude API 调用示例和 JSON Schema 备选方案 |
