## 整体架构

页面是一个 **Next.js 16 App Router + shadcn/ui + Tailwind CSS** 构建的静态站点，部署在 Cloudflare Pages。纯服务端渲染（SSG），数据来自 `data/exports/*.md` 文件，无需数据库。

### 路由结构

| 路由 | 页面 | 组件 |
|------|------|------|
| `/` | 首页 | `page.tsx` |
| `/date/[date]` | 日期页 - 某天全部新闻 | `date/[date]/page.tsx` |
| `/news/[slug]` | 新闻详情页 - 完整正文 + 5 个论文选题 | `news/[slug]/page.tsx` |
| `/category/[field]` | 分类页 - 按经济领域筛选 | `category/[field]/page.tsx` |
| `/archive` | 归档页 - 按月份聚合 | `archive/page.tsx` |

### 组件清单

| 组件 | 用途 |
|------|------|
| `NewsCard.tsx` | 新闻摘要卡片 + 折叠选题预览 |
| `TopicCard.tsx` | 论文选题卡片（含研究问题/理论框架/方法/数据/创新点表格） |
| `FilterBar.tsx` | 经济领域分类筛选标签栏 |
| `Badge.tsx` | 领域分类彩色标签（含 `FieldBadge` 和 `FieldBadgeSm` 两个变体） |
| `Breadcrumb.tsx` | 面包屑导航 |

## 4. 主内容区 — 按日期分组新闻卡片列表

这是页面的核心，按**日期降序**排列（最新在前）。

### 日期标题行

```
📅 2026-06-05  ← 可点击跳转到 /date/2026-06-05
6 条新闻 · 30 个选题
```

- 左侧琥珀色边框 + 渐变背景
- hover 时日期文字变色
- **源码位置**：CSS `.date-heading`，`src/app/page.tsx` 第 38-48 行

### 新闻卡片结构（NewsCard）

```
┌──────────────────────────────────────────────┐
│ 🏭 产业经济                    2026-06-05    │  ← 领域标签 + 日期
│                                              │
│ 各地麦收高效推进 助力夏粮颗粒归仓             │  ← 新闻标题（h2）
│ 原文链接 →                                   │  ← 原文链接（gov.cn）
│                                              │
│ 当前，我国各地加快冬小麦收获进度…（摘要截断）  │  ← 150字摘要，3行clamp
│                                              │
│ ▶ 🎓 5 个论文选题                             │  ← details/summary 折叠
│   1. 新古典视角：跨区农机调度对收割技术效率…   │
│   2. 凯恩斯与新凯恩斯视角：…                   │
│   …（展开后可见 5 个）                         │
└──────────────────────────────────────────────┘
```

**卡片元素详解**：
- **领域标签**：彩色 badge（`FieldBadgeSm`）+ 日期
- **新闻标题**：h2 元素，点击进入 `/news/[slug]` 详情页，hover 变琥珀色
- **原文链接**：带下滑线动画（`background-size` 过渡），跳转 gov.cn 原文
- **正文摘要**：限 150 字符，CSS 3 行 clamp 截断
- **折叠选题列表**：HTML `<details>` 元素，默认折叠，展开后以有序列表展示 5 个选题（视角 + 标题）

**卡片样式**：
- 白色背景 + `box-shadow: var(--shadow-sm)`
- hover 时阴影提升至 `shadow-md` + 边框变深
- 卡片间距 0.85rem
- **源码位置**：`src/components/NewsCard.tsx`

### 数据流

```
data/exports/*-paper-topics.md
    ↓ fs.readFileSync
src/lib/parser.ts (parseDailyExport)
    ↓ 解析 frontmatter + Markdown 结构
src/lib/data.ts (getAllDates / getByDate)
    ↓ 内存缓存 60 秒
src/app/page.tsx (HomePage)
    ↓ 渲染
页面输出
```

**解析器能力**：
- 解析 YAML frontmatter（日期、领域、新闻数、选题数、生成时间）
- 解析每条新闻的标题、URL、领域分类、正文内容
- 解析 5 个论文选题（视角、标题、研究问题、理论框架、研究方法、数据来源、创新点）
- 自动归一化经济领域 emoji 前缀
- 从 `content_XXXXXXX.htm` URL 提取 slug

**源码位置**：`src/lib/parser.ts`（355 行）、`src/lib/data.ts`（136 行）

## 9. 待优化方向

基于以上分析，以下是可调整的方向：

### 导航与信息架构
1. **导航栏过于精简** — 只有「首页」，归档页虽存在但无入口；可加回归档链接或增加「关于」页
2. **无搜索功能** — 选题和新闻增多后无法搜索，可考虑添加客户端搜索或静态搜索索引

### 首页体验
3. **Hero 区缺少行动引导** — 可增加「查看今日选题 ↓」按钮或统计数字展示（总选题数、覆盖天数）
4. **分类筛选无计数** — FilterBar 只显示分类名，可显示各分类下新闻数量
5. **卡片列表随内容增长变长** — 多天数据堆积后需大量滚动，可考虑分页、按月份折叠、或默认只显示最近 N 天

### 内容呈现
6. **首页可增加侧边栏或时间线导航** — 快速跳转到指定日期
7. **缺少数据可视化** — 可增加领域分布图、选题趋势等

### 页脚与辅助
8. **Footer 信息略显单薄** — 可增加关于页链接、RSS 订阅、GitHub 仓库链接等
9. **缺少 SEO 增强** — 可增加 sitemap、结构化数据（Schema.org）

### 功能扩展
10. **无选题收藏/标注功能** — 研究者可能需要标记感兴趣的选题
11. **无导出功能** — 可考虑支持选题导出为 BibTeX 或 PDF
