# EconTopic 自动化方案

## 核心问题

> 如何将抓取 → 筛选 → 分析 → 发布这 4 个步骤串联，实现每日自动运行？

---

## 两层架构总览

```
┌─────────────────────────────────────────────┐
│  Claude Code 层（开发 & 调试用）              │
│  /scrape-gov  /filter-economic  /daily-run  │
│  economic-topic-generator (Agent)            │
│  → 手动触发，调试 Prompt，验证质量            │
└──────────────┬──────────────────────────────┘
               │ 调试通过后，核心逻辑固化为代码
               ▼
┌─────────────────────────────────────────────┐
│  生产层（自动化运行）                         │
│  Vercel Cron Job 每日定时触发                │
│  → API Route 串联 pipeline                  │
│  → 写入数据库 → 网站自动展示                  │
└─────────────────────────────────────────────┘
```

**Skill/Agent 帮你「造出系统」，而不是系统本身。系统造好后，自动运行靠的是代码 + 定时调度器。**

---

## 路线 A：Meta-Skill + Workflow 编排（推荐长期使用）

### 设计思路

创建一个 `/daily-pipeline` Skill，内部用 **Claude Code Workflow** 编排多个 Agent，实现全流程自动化。然后用 `/loop` 定时触发。

### Skill 结构

```
/daily-pipeline
  │
  ├── Phase 1: 抓取
  │     └── agent("抓取 gov.cn 今日新闻", agentType: "general-purpose")
  │
  ├── Phase 2: 筛选
  │     └── agent("从抓取结果中筛选经济相关新闻", agentType: "general-purpose")
  │
  ├── Phase 3: 分析（核心——并发 pipeline）
  │     └── pipeline(经济新闻列表,
  │           news => agent(`分析这篇新闻的论文选题: ${news.title}`),
  │           analysis => agent(`质检: ${analysis}`)  ← topic-quality-reviewer
  │         )
  │
  └── Phase 4: 发布
        └── agent("将分析结果写入数据库并刷新网站")
```

### 完整 Workflow 脚本

```javascript
export const meta = {
  name: 'daily-pipeline',
  description: '一键全流程：抓取 gov.cn → 筛选经济新闻 → 生成论文选题 → 发布网站',
  phases: [
    { title: '抓取', detail: '爬取中国政府网今日新闻' },
    { title: '筛选', detail: '筛选经济相关新闻' },
    { title: '分析', detail: '生成论文写作选题' },
    { title: '质检', detail: '审核分析质量' },
    { title: '发布', detail: '写入数据库并刷新网站' },
  ],
};

// ===== Phase 1: 抓取 =====
phase('抓取');
const scrapedNews = await agent(
  `访问 https://www.gov.cn，抓取今日最新发布的新闻列表。
   提取每条的：标题、发布时间、正文摘要、原文URL。
   以 JSON 数组格式返回。`,
  { label: '抓取gov.cn', schema: NEWS_LIST_SCHEMA }
);

// ===== Phase 2: 筛选 =====
phase('筛选');
const economicNews = await agent(
  `从以下新闻中筛选出与经济相关的条目。经济相关包括：
   宏观经济、财政政策、货币政策、产业经济、金融、贸易、
   区域发展、就业、物价、房地产、数字经济、绿色经济。
   
   新闻列表：${JSON.stringify(scrapedNews)}
   
   返回经济相关新闻，并标注所属子领域和相关度(高/中/低)。`,
  { label: '经济筛选', schema: ECONOMIC_NEWS_SCHEMA }
);

// ===== Phase 3: 分析 + 质检（并发 pipeline）=====
const analyses = await pipeline(
  economicNews.batch,
  (news) => agent(
    `你是一个经济学教授。基于以下新闻，生成 1-3 个可以作为
     经济学论文写作方向的话题。对每个话题给出：
     - 论文标题建议
     - 核心研究问题
     - 可用理论框架（新古典/凯恩斯/制度/行为等视角）
     - 研究方法建议
     - 可能的数据来源
     - 创新点
     
     新闻：${news.title}
     正文：${news.content}`,
    { phase: '分析', schema: PAPER_TOPICS_SCHEMA }
  ),
  (analysis) => agent(
    `严格审核以下论文选题的质量：
     1. 是否足够具体，而非泛泛而谈？
     2. 理论框架是否匹配？
     3. 数据来源是否真实可获取？
     4. 创新点是否站得住脚？
     返回 { pass: true/false, score: 1-5, feedback: "..." }`,
    { phase: '质检', schema: QUALITY_CHECK_SCHEMA }
  )
);

// ===== Phase 4: 发布 =====
phase('发布');
const publishResult = await agent(
  `将以下 ${analyses.length} 条分析结果写入数据库，
   触发网站首页重新生成。
   数据库连接信息从 .env.local 读取。`,
  { label: '发布到网站' }
);

return {
  date: new Date().toISOString().split('T')[0],
  scraped: scrapedNews.length,
  economic: economicNews.length,
  analyzed: analyses.length,
  published: true,
};
```

### 每日自动触发

```bash
# 基础方式 — 每天定时跑一次
/loop 1d /daily-pipeline

# 持久化 — 电脑重启后仍然存活
/loop 1d /daily-pipeline --durable
```

## 路线 B：单一自治 Agent（推荐快速验证用）

### 设计思路

创建一个 `econtopic-automator` Agent，把全流程压缩到一段系统提示中。Agent 内部自主完成所有步骤。

### Agent 系统提示

```
你是一个自动化经济新闻分析系统，名为 EconTopic Automator。

## 你的任务流程

每次被调用时，按顺序执行以下步骤：

1. **抓取** — 使用 Playwright 访问 https://www.gov.cn，抓取今日最新发布的新闻。
   提取每条的标题、发布时间、正文摘要和原文 URL。

2. **筛选** — 从所有新闻中筛选出与经济相关的条目。经济相关包括：
   - 宏观经济（GDP、经济增长、经济形势）
   - 财政政策（预算、税收、国债、转移支付）
   - 货币政策（利率、准备金、信贷、货币供应）
   - 产业经济（制造业、数字经济、平台经济、能源转型）
   - 金融（银行、证券、保险、外汇、跨境资本）
   - 国际贸易（关税、贸易协定、一带一路、供应链）
   - 区域经济（城镇化、乡村振兴、区域协调、地方债务）
   - 就业与物价（CPI、失业率、收入分配）

3. **分析** — 对每条经济新闻，以经济学教授视角生成 1-3 个论文写作方向。
   每个方向必须包含：
   - 论文标题建议
   - 核心研究问题（1-2 句）
   - 可用理论框架
   - 研究方法建议
   - 可能的数据来源
   - 创新点

4. **质检** — 审核每个选题：
   - 是否足够具体？
   - 理论框架是否匹配？
   - 数据来源是否可获取？
   - 创新点是否站得住脚？
   评分 1-5，低于 3 分的选题丢弃。

5. **发布** — 将合格的分析结果写入数据库，刷新网站首页。

## 输出格式

任务完成后，返回执行报告：
{
  "date": "YYYY-MM-DD",
  "scraped_count": N,
  "economic_count": N,
  "analyzed_count": N,
  "published_count": N,
  "topics": [...]
}
```

### 触发方式

```
# 创建 Skill 包装这个 Agent
/daily-auto

# 设置定时
/loop 1d /daily-auto
```

或者更简单，直接在 `/loop` 中用自然语言触发：

```bash
/loop 1d 执行 econtopic-automator 的完整流程：抓取、筛选、分析、发布
```

## 两条路线对比

| 维度 | 路线 A: Workflow 编排 | 路线 B: 单 Agent |
|---|---|---|
| **可控性** | ⭐⭐⭐⭐⭐ 每步可单独调试、重跑 | ⭐⭐ 黑盒运行，出错难定位 |
| **可靠性** | 每步独立，失败不丢上下文 | 单点故障风险 |
| **并行能力** | `pipeline()` 并发分析，速度快 | 顺序执行，耗时长 |
| **可观测性** | 每阶段有进度条和独立日志 | 只有最终输出 |
| **维护成本** | 需维护 Workflow 脚本 | 只需维护一段系统提示 |
| **Token 成本** | 每步独立 Agent，总 token 较高 | 单 Agent，token 较低 |
| **适合场景** | 长期运行，每日稳定产出 | 快速验证，灵活调整 |

## 推荐实施策略

```
第 1 阶段（本周）: 路线 B 单 Agent 原型
  └── 目标：2 小时内跑通全流程，看到第一条分析结果
  └── 验证：抓取可行？筛选准确？分析有价值？

第 2 阶段（验证满意后）: 迁移到路线 A
  └── 目标：分拆为独立 Agent，用 Workflow 编排
  └── 验证：并行分析速度、质检过滤效果

第 3 阶段（稳定运行后）: 考虑 Vercel Cron Job
  └── 目标：7×24 无人值守，不依赖 Claude Code 运行
  └── 验证：连续运行一周无故障
```

## 自动化的关键限制

| 限制 | 说明 |
|---|---|
| Claude Code 必须运行 | `/loop` 依赖 Claude Code 进程存活，电脑关机则停止 |
| 持久化需显式指定 | `--durable` 参数将任务写入 `.claude/scheduled_tasks.json` |
| 补跑机制 | Claude Code 重启后会自动检查过期任务并补跑 |
| 真·无人值守 | 如需 7×24 运行，需要部署到 Vercel Cron Job（见下方） |

## 终极方案：Vercel Cron Job（7×24 无人值守）

当项目成熟后，将核心逻辑固化为 Next.js API Route，用 Vercel Cron Jobs 在生产环境运行：

```typescript
// app/api/cron/daily/route.ts
export async function GET(request: Request) {
  const authHeader = request.headers.get('authorization');
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return new Response('Unauthorized', { status: 401 });
  }

  const news = await scrapeGovNews();           // Step 1
  const economicNews = await filterEconomic(news); // Step 2
  const analyses = await Promise.all(              // Step 3
    economicNews.map(n => generatePaperTopics(n))
  );
  await db.analyses.createMany({ data: analyses }); // Step 4 写入DB
  revalidatePath('/');                               // Step 4 刷新首页

  return Response.json({ scraped: news.length, analyzed: analyses.length });
}
```

```json
// vercel.json
{
  "crons": [
    {
      "path": "/api/cron/daily",
      "schedule": "0 0 * * *"   // UTC 00:00 = 北京时间 08:00
    }
  ]
}
```

## 各组件在自动化中的角色总结

| 组件 | 类型 | 在自动化中的角色 |
|---|---|---|
| `/scrape-gov` | Skill | 开发期调试爬虫逻辑，产出 `scrapeGovNews()` 函数 |
| `/filter-economic` | Skill | 开发期建立关键词库，产出 `filterEconomic()` 函数 |
| `/analyze-paper-topics` | Skill | 开发期打磨 Prompt，产出 `generatePaperTopics()` 函数 |
| `gov-page-analyzer` | Agent | 线上爬虫失败时，派遣排查页面结构变化 |
| `economic-topic-generator` | Agent | 被 Workflow 并发调用，每条新闻生成选题 |
| `topic-quality-reviewer` | Agent | 被 Workflow pipeline 调用，审核分析质量 |
| `/daily-pipeline` | Skill | Workflow 串联全流程，开发期 + 半自动期的主入口 |
| `/loop 1d` | CLI 命令 | 定时触发器，让 `/daily-pipeline` 每天自动跑 |
| `api/cron/daily` | API Route | 生产环境自动化入口，被 Vercel Cron Job 触发 |
