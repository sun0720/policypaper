# 新闻联播数据源接入方案

## 目标

在现有 PolicyPaper 系统（中国政府网 → 经济论文选题）的基础上，新增**新闻联播**作为第二个新闻数据源：
- 从新闻联播网站抓取每日新闻
- 筛选与经济相关的新闻
- 基于经济新闻生成论文选题
- 发布到网站展示

---

## 一、整体策略：复用现有管道，新增抓取层

现有 `economic-filter`、`paper-topic-analyzer` 和 `site-publisher` 三个模块都是**与新闻源无关的**——它们只消费 JSON 格式的新闻数据，不关心新闻来自哪个网站。因此 75% 的工作已完成。

需要做的核心工作：
1. **新建 `cctv-scraper` Skill**（对标 `gov-scraper`）
2. **调整数据流和网站展示**以支持多源

## 二、Step 1 调研结果：新闻联播网站页面结构

> 调研日期：2026-06-10
> 目标网站：https://tv.cctv.com/lm/xwlb/

### 2.1 核心发现

| 项目 | 结论 |
|---|---|
| 页面渲染方式 | **服务端渲染 HTML**（非 JS 动态加载），无需无头浏览器 |
| 列表 CSS 选择器 | `ul#content.rililist.newsList > li` |
| 日期 URL 模式 | `https://tv.cctv.com/lm/xwlb/day/YYYYMMDD.shtml` |
| 默认页面 | 显示最新一期（当天或最近一期）的新闻片段 |
| API 可用性 | `api.cntv.cn/NewVideo/getVideoListByColumnId` → **拒绝访问（1100）**；`api.cntv.cn/lanmu/columnInfoByColumnId` → 可用但仅有栏目元数据 |
| ⚠️ 正文文本 | **视频页面无文字稿（transcript）**，仅有标题 + 简短 meta description |

### 2.2 页面 HTML 结构

```html
<ul id="content" class="rililist newsList">
  <!-- 第一条：完整版（整期节目） -->
  <li>
    <div class="image">
      <a href="https://tv.cctv.com/2026/06/09/VIDExxx.shtml"
         alt="《新闻联播》 20260609 21:00"
         title="《新闻联播》 20260609 21:00">
        <img src="..." />
      </a>
      <span></span>
    </div>
    <a href="...">
      <i class="sql0">完整版</i>《新闻联播》 20260609 21:00
    </a>
  </li>
  <!-- 后续条目：单条新闻片段 -->
  <li>
    <div class="image">
      <a href="https://tv.cctv.com/2026/06/09/VIDEyxxx.shtml"
         alt="[视频]前5个月我国货物贸易进出口同比增长15.3%"
         title="[视频]前5个月我国货物贸易进出口同比增长15.3%">
        <img src="..." />
      </a>
      <span></span>
    </div>
    <a href="...">
      <i class="sql1">完整版</i>[视频]前5个月我国货物贸易进出口同比增长15.3%
    </a>
  </li>
  ...
</ul>
```

### 2.3 两种条目类型

| 类型 | CSS 标识 | 含义 | 抓取策略 |
|---|---|---|---|
| 完整版 | `<i class="sql0">完整版</i>` | 整期新闻联播节目视频 | **跳过** — 无独立标题，无文字稿 |
| 单条片段 | `<i class="sql1">完整版</i>` | 单条新闻的视频片段 | **抓取** — 有独立标题，用于经济筛选 |

### 2.4 可提取的数据字段

从列表页 HTML 直接提取：

| 字段 | 提取来源 | 示例 |
|---|---|---|
| `title` | `<a>` 的 `alt` 属性（去掉 `[视频]` 前缀） | `前5个月我国货物贸易进出口同比增长15.3%` |
| `url` | `<a>` 的 `href` 属性 | `https://tv.cctv.com/2026/06/09/VIDEzxxx.shtml` |
| `date` | URL 中的日期路径 `/2026/06/09/` | `2026-06-09` |
| `thumbnail` | `<img>` 的 `src` 属性 | `//p1.img.cctvpic.com/.../xxx.jpg` |

### 2.5 ⚠️ 与 gov-scraper 的关键差异：无正文

**新闻联播的视频页面不提供文字稿（transcript）。** 视频详情页仅为视频播放器，仅有：
- 标题（`<meta name="description">`）
- 关键词（`<meta name="keywords">`）
- 无正文 `content` 字段

这意味着：
- `economic-filter` 只能基于**标题**进行分类（准确率略低于标题+正文，但仍可工作）
- `paper-topic-analyzer` 只能基于**标题**生成选题（选题会更依赖 LLM 的背景知识补充）
- 输出的 `content` 字段将为空字符串

### 2.6 抓取策略总结

```
1. HTTP GET → https://tv.cctv.com/lm/xwlb/day/{YYYYMMDD}.shtml
2. 正则提取 <ul id="content" class="rililist newsList"> 中的所有 <li>
3. 过滤掉 <i class="sql0">（完整版）
4. 对每个 <i class="sql1">，提取 title/url/date
5. 输出 JSON → data/raw/cctv/{date}.json
```

**优点**：零依赖（Python 标准库即可），无需无头浏览器，速度快（1 次 HTTP 请求）。
**缺点**：无正文文本，下游分析仅靠标题。

## 三、新建 `cctv-scraper` Skill

完全对标 `gov-scraper` 的结构，位于 `.claude/skills/cctv-scraper/`：

```
.claude/skills/cctv-scraper/
├── SKILL.md                         ← Skill 定义
├── scripts/scrape_cctv.py           ← 核心抓取脚本
└── references/page-structure.md     ← 新闻联播网站页面结构文档（即本调研结果）
```

### 与 gov-scraper 的关键差异

| 差异点 | gov-scraper | cctv-scraper |
|---|---|---|
| 数据源 | JSON API (`YAOWENLIEBIAO.json`) | HTML 页面（服务端渲染） |
| 解析方式 | `json.load()` | `re.findall()` / `BeautifulSoup` |
| 正文获取 | `--content` 深入抓取原文 | **不支持**（无文字稿） |
| 完整版/分条 | 无此区分 | 需跳过 `sql0` 完整版，只取 `sql1` 片段 |
| 日期格式 | `YYYY-MM-DD` | 日期从 URL 路径 `/YYYY/MM/DD/` 提取 |
| 条目数 | 每天 0-30 条（含所有类型新闻） | 每天 10-20 条单条片段 |
| 经济新闻占比 | 约 15-25% | 约 10-20%（偏政治/外交，经济占比更低） |

### 输出 JSON 格式

```json
[
  {
    "title": "前5个月我国货物贸易进出口同比增长15.3%",
    "url": "https://tv.cctv.com/2026/06/09/VIDEzAEu80lkby0LvlxVHzn8260609.shtml",
    "date": "2026-06-09",
    "content": "",
    "thumbnail": "//p1.img.cctvpic.com/photoAlbum/vms/standard/img/2026/6/9/VIDE6pNn5isDhpPwqZN9vZI1260609.jpg",
    "source": "cctv"
  }
]
```

> ⚠️ `content` 字段为空（无文字稿）。`economic-filter` 和 `paper-topic-analyzer` 需适配仅标题输入的场景。

## 三、数据目录结构调整

从单源变为双源，数据目录体现来源：

```
data/
├── raw/
│   ├── gov/                        ← 中国政府网（原有）
│   │   ├── 2026-06-10.json
│   │   └── ...
│   └── cctv/                       ← 新闻联播（新增）
│       ├── 2026-06-10.json
│       └── ...
├── filtered/
│   ├── gov/
│   │   └── 2026-06-10.json
│   └── cctv/
│       └── 2026-06-10.json
├── exports/
│   ├── gov/
│   │   └── 2026-06-10.md
│   └── cctv/
│       └── 2026-06-10.json
└── .cache/
    └── parsed.json
```

**核心原则**：raw 和 filtered 层分源存储，exports 层合并输出。网站展示时用户关注的是选题内容而非来源。

## 六、Pipeline 调整（核心改动）

### 新的双源流水线架构

```
                    ┌─ 🔍 Scrape-gov ────→ data/raw/gov/{date}.json
                    │
  🚀 Pipeline 触发 ──┤
                    │
                    └─ 🔍 Scrape-cctv ───→ data/raw/cctv/{date}.json
                            │
                    ┌───────┘
                    ▼
              🏷️ Filter-gov ──→ data/filtered/gov/{date}.json
              🏷️ Filter-cctv ─→ data/filtered/cctv/{date}.json
                    │
                    ▼
              🧩 合并两个 filtered JSON（按标题相似度去重）
                    │
                    ▼
              🧠 Analyze（并行逐条生成选题，每条新闻 1 个 agent）
                    │
                    ▼
              🧩 Combine → data/exports/{date}.md
                    │
                    ▼
              🚀 Publish → 网站部署
```

### 关键改动点

1. **Phase 1** 拆为两个并行的 agent（Scrape-gov + Scrape-cctv），各自独立抓取
2. **Phase 2** 同样并行过滤
3. 新增**合并步骤**：将两个 filtered JSON 合并为一个统一的输入文件
4. **去重逻辑**：两个源的新闻可能有重复（同一天同一政策两边都报道），合并时按标题相似度 >80% 视为重复，保留来源更权威的那条
5. Phase 3-5 不变，因为下游模块是源无关的

## 九、关键风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| ~~反爬/需登录~~ | ~~无法抓取~~ | ✅ 已确认：服务端渲染 HTML，无反爬，Python 标准库即可抓取 |
| ⚠️ **视频页面无文字稿** | `content` 字段为空，`economic-filter` 和 `paper-topic-analyzer` 只能基于标题工作 | 标题通常含足够信息（如「前5个月我国货物贸易进出口同比增长15.3%」）；`economic-filter` 的 Prompt 中注明「仅标题模式」；选题生成质量可能略降 |
| 新闻联播内容偏政治/外交，经济新闻占比低 | 每天仅 1-3 条经济新闻 | 预期合并 gov.cn 后总量 5-10 条，够用；若太少可考虑再接入其他源 |
| 两个源新闻重复（同一政策两边都报道） | 选题重复 | 合并阶段按标题相似度去重（>80% 视为重复，保留 gov.cn 的（有正文更丰富）） |
| 新闻联播标题风格（`[视频]` 前缀、播音稿式概括） | 经济分类准确率下降 | 抓取时去除 `[视频]` 前缀；`economic-filter` Prompt 中注明「新闻联播标题模式」 |
| API 调用成本增加 | 月度费用上升 | 每天多 1-3 条经济新闻（且无正文 = token 更少），实际增量有限；预估从 $30-60/月 升至 $40-70/月 |
