# PolicyPaper — 中国经济新闻 → 论文话题生成系统

## 项目概述

每日自动抓取**中国政府网 (www.gov.cn)** 最新新闻，提取经济相关新闻点，利用 LLM 分析可作为经济学论文写作话题的方向，最后将分析结果发布到网站上。

---

## 一、系统整体架构

```
中国政府网 ──→ 爬虫模块 ──→ 经济过滤 ──→ LLM分析 ──→ 网站展示
  (gov.cn)      (每日)       (关键词+AI)   (话题生成)   (前端页面)
```

### 数据流

```
1. Cron Job 触发（每日定时）
2. 爬虫抓取 gov.cn 当日新闻列表
3. 关键词初筛 → LLM 二次分类 → 经济相关新闻
4. LLM 深度分析 → 生成论文话题
5. 结构化数据存入数据库
6. 网站前端展示、按日期/标签归档
```

## 二、建议创建的核心 Claude Code Skills

### 1. `gov-scraper` — 政府网新闻抓取

**职责**：每日自动爬取 www.gov.cn 最新发布的新闻

**核心挑战**：
- 中国政府网可能使用 Cloudflare / 动态渲染等反爬保护，需 Playwright / Puppeteer 无头浏览器方案
- HTML 结构复杂，需精确的 CSS 选择器定位
- GBK/UTF-8 编码处理
- 日期格式统一与去重

**Skill 封装的能力**：

| 能力项 | 说明 |
|---|---|
| 初始化项目结构 | Python + Playwright 或 Node + Puppeteer 脚手架 |
| 自动发现 URL 模式 | 分析 gov.cn 新闻列表页的分页/分类 URL 规律 |
| 结构化提取 | 标题、发布时间、正文、原文链接、来源部门 |
| 增量抓取 | 日期比对 + URL 去重，不重复爬已有新闻 |
| 异常处理 | 网络超时重试、验证码/封IP检测、日志记录 |

---

### 2. `economic-filter` — 经济新闻过滤与分类

**职责**：从海量政务新闻中精准筛出经济相关内容

**核心挑战**：
- 中国政府网内容覆盖面极广（政治、外交、民生、军事、科技、文化...），经济类仅占约 15-25%
- 经济学术语在政务语境下的中文表达差异大（如「稳住经济大盘」= 宏观稳定政策）
- 某些表面是「会议报道」的新闻，实际涉及重大经济决策，需要语义理解

**经济分类体系**：

```
📊 宏观经济
  ├── GDP 增长、经济形势
  ├── 财政政策（预算、税收、国债）
  ├── 货币政策（利率、准备金、信贷）
  └── 就业与物价（CPI、失业率）

🏭 产业经济
  ├── 制造业与工业政策
  ├── 数字经济与平台经济
  ├── 能源与绿色转型
  └── 房地产与基建

💰 金融
  ├── 银行与信贷
  ├── 证券市场（股市、债市）
  ├── 保险与养老金
  └── 外汇与跨境资本

🌏 国际贸易
  ├── 关税与贸易协定
  ├── 一带一路
  ├── 供应链与产业链
  └── 外资与对外投资

🏘️ 区域经济
  ├── 城镇化与城市群
  ├── 乡村振兴
  ├── 区域协调发展
  └── 地方债务
```

**Skill 封装的能力**：

| 能力项 | 说明 |
|---|---|
| 关键词三级库 | 维护中文经济关键词库，按子领域分类，持续更新 |
| 初筛管道 | 基于关键词匹配 + 正则规则，快速过滤明显不相关的新闻 |
| LLM 精准分类 | 将初筛结果送 LLM，按经济学子领域打标签，标记相关度（高/中/低） |
| 假阴性回溯 | 定期人工抽样检验漏网之鱼，反哺关键词库 |

### 3. `paper-topic-analyzer` — 经济学论文话题生成 ⭐

**职责**：对每条经济新闻，生成 1-3 个可落地写作的经济学论文方向

> ⚠️ 这是整个系统的**价值核心**，产出质量直接决定网站竞争力。

**核心挑战**：
- 需要产出**专业级**的经济学分析，而非泛泛的新闻总结
- 需要结合经济学理论框架，建立新闻事件与学术研究的桥梁
- 提示词设计极为关键，迭代打磨是主要工作量

**标准化输出格式**：

```json
{
  "news_title": "原文标题",
  "news_summary": "100字以内的新闻摘要",
  "economic_field": "宏观经济学",
  "paper_topics": [
    {
      "title": "论文标题建议",
      "research_question": "核心研究问题（1-2句）",
      "theoretical_framework": "可用的理论框架（如 IS-LM、新凯恩斯DSGE、制度变迁理论等）",
      "methodology_hints": "建议的研究方法（实证/理论/案例研究，具体回归方法等）",
      "data_sources": "可能的数据来源（国家统计局、Wind、CSMAR、CGSS等）",
      "innovation_point": "该话题的创新点或独特视角",
      "reference_hint": "可参考的经典文献方向"
    }
  ]
}
```

**多视角分析 Prompt 设计**：

| 视角 | 分析重点 |
|---|---|
| 新古典视角 | 市场均衡、价格信号、效率与福利 |
| 凯恩斯视角 | 总需求管理、乘数效应、财政/货币政策传导 |
| 制度经济学 | 产权、交易成本、制度变迁、路径依赖 |
| 行为经济学 | 非理性行为、心理偏差、政策的微观行为基础 |
| 政治经济学 | 利益集团、政策博弈、央地关系 |
| 发展经济学 | 结构转型、人力资本、技术追赶、中等收入陷阱 |

**Skill 封装的能力**：

| 能力项 | 说明 |
|---|---|
| 多视角 Prompt 模板 | 维护一套可配置的分析 Prompt，支持不同经济学流派视角 |
| 结构化输出 | 通过 Claude API + JSON Schema 确保输出格式统一 |
| 质量自检 | 二次 LLM 调用，对生成结果打分：是否足够具体？是否可操作？是否有论文价值？ |
| 批量处理 | 一次 API 调用分析多条新闻，降低 token 成本 |
| Prompt 迭代工具 | A/B 测试框架，对比不同 Prompt 产出的质量 |

### 4. `site-publisher` — 网站发布与展示

**职责**：将分析结果发布到网站，提供良好阅读体验

**核心需求**：

| 功能 | 说明 |
|---|---|
| 首页 | 最新分析列表，按日期降序 |
| 详情页 | 单条新闻 + 全部论文话题展开 |
| 分类筛选 | 按经济领域标签过滤 |
| 日期归档 | 按日/周/月浏览历史 |
| 搜索 | 全文搜索新闻和分析内容 |
| 暗色模式 | 阅读友好 |
| RSS/订阅 | 可选，邮件/微信推送 |

**推荐技术栈**：

```
前端：Next.js 16 + shadcn/ui + Tailwind CSS
后端 API：Next.js API Routes / Server Actions
数据库：Vercel Postgres（Neon）或 Supabase
定时任务：Vercel Cron Jobs
部署：Vercel（一键部署、自动 SSL、全球 CDN）
AI 服务：Claude API（经济分析）+ 可选 OpenAI 备份
```

**Skill 封装的能力**：

| 能力项 | 说明 |
|---|---|
| 双渲染策略 | MVP（MDX 文件直达，免数据库）和生产（Vercel Postgres + 搜索/筛选）两种方案 |
| 数据库 Schema | news + topics 两表设计，含全文搜索索引、幂等 Upsert 导入 |
| 页面模板 | 首页、日期页、详情页（选题卡片）、分类筛选、归档、搜索共 6 种页面 |
| 学术排版体系 | 衬线标题 + 1.8 正文行高 + 暖白纸张底色 + 琥珀棕强调色 |
| 数据导入脚本 | `import-data.ts`，支持 JSON/MD 双格式，`--all`/`--date`/`--dry-run` |
| 一键部署 | 通过 Vercel 部署 + Cron Jobs 实现每日自动更新 |

> 📁 Skill 位置：`.claude/skills/site-publisher/`（已于 2026-06-05 创建）
> 📄 详见：`CLAUDE.md` → `# 核心模块` → `4. site-publisher`

## 五、关键风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| gov.cn 反爬严格 | 项目无法启动 | Playwright 无头浏览器 + 代理 IP 池 + 降低抓取频率 |
| LLM 分析质量不稳定 | 产品价值低 | 多轮 Prompt 迭代 + 人工抽检 + 用户反馈闭环 |
| API 调用成本过高 | 不可持续 | 日摘要模式（多条新闻合并一次分析）+ 缓存 + 分级分析 |
| 版权/合规风险 | 法律问题 | 仅使用标题和公开摘要，注明来源链接，不全文转载 |
