# 提示优化分析

## 原始提示

> 我是一个经济学家，我想制造一个网站。这个网站的作用是，每天抓取《中国政府网》上的最新新闻，然后提取与经济相关的新闻点，分析这些新闻有哪些点可以作为经济话题进行经济学论文的写作，最后将生成的写作分析点发不到这个网站上。

## 识别的问题

### 清晰度
- 「制造一个网站」表述模糊 — 是静态博客？内容平台？单页应用？
- 「分析」的核心方式未定义 — AI 自动分析？人工撰写？基于规则匹配？
- 「发不到」为笔误（应为「发布到」），发布形式未说明

### 具体性
- 未指定前端框架
- 中国政府网的抓取方式未说明（RSS？API？网页爬虫？）
- AI 分析模型未选定
- 数据存储方案未说明
- 部署方式未说明（现已有：GitHub + Cloudflare Pages）

### 完整性
- 缺少成功标准（每天处理几篇？分析多深？）
- 缺少内容展示形式
- 缺少定时触发的具体方案（Cloudflare Pages 本身无 Cron，需外部触发）

### 结构问题
- 全部要求混在一个段落里，没有分步骤或分层

### 可操作性
- 「制造一个网站」过于宽泛，缺乏可拆解的具体交付物
- 没有分阶段实施计划

## 优化提示

### 构建一个「中国经济政策论文选题生成器」网站

**项目概述**：自动抓取中国政府网（www.gov.cn）每日新闻，用 AI 筛选经济相关条目并分析可研究的论文选题方向，将结果发布到网站上供学者参考。

---

### 一、技术架构
| 层面 | 方案 |
|------|------|
| 前端框架 | Next.js / Astro / Hugo（待选定，输出为静态站） |
| 部署方式 | GitHub 仓库 → Cloudflare Pages 自动构建部署 |
| 定时任务 | Cloudflare Workers Cron Trigger 或 GitHub Actions 定时触发 |
| 数据抓取 | Node.js 脚本或 Python 脚本，爬取中国政府网新闻 |
| AI 分析 | 调用大模型 API（Claude / GPT / DeepSeek）分析经济学选题 |
| 数据存储 | Cloudflare D1（SQLite）或 KV，存储历史分析结果 |

### 二、核心流程

```
定时抓取（每日一次）
    ↓
经济筛选（关键词 + AI 判断）
    ↓
选题分析（AI 生成论文选题方向）
    ↓
内容发布（静态网站展示）
```

1. **定时抓取**（每日一次）：爬取 www.gov.cn 当日发布的新闻列表
2. **经济筛选**：用关键词 + AI 判断，筛选与经济政策相关的新闻
3. **选题分析**：AI 对每条经济新闻生成论文选题方向，包含：
   - 选题方向建议
   - 理论切入点（宏观/微观/国际经济学/产业经济学等）
   - 可能的数据来源
4. **内容发布**：生成的选题分析发布到静态网站，可按日期、分类浏览

### 三、功能需求

**MVP（最小可行产品）**
- 按日期展示每日选题分析列表
- 按经济学分支分类筛选（货币、财政、贸易、产业、劳动等）
- 支持搜索历史选题

**可选增强**
- 数据可视化展示（关键词趋势、政策频率等）
- 邮件/RSS 订阅每日推送
- 用户评论/讨论功能
- 论文选题收藏与导出

### 四、待用户确认

- [ ] AI 模型选择（Claude / GPT / 国内模型）
- [ ] 是否公开访问 or 个人使用？
- [ ] 分析深度：每日摘要级 or 深度论文级？
